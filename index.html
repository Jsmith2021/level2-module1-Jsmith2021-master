Rain Catcher

by: Jessica Smith

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I decided to make this game because I wanted to draw peoples attention to California's drought and the importance of saving water. The object of my game is to collect as many inches of water as possible and this object encourages people to save water in California's drought.

2. What I Learned

From user testing, I learned how important it is to listen and collaborate with my peers and teacher to better my game. My teachers and classmates suggested I make the bucket move more smoothly, make the rock and rain move at random speeds, and to make the bucket not move off the page.

3. Architecture of My Game

My game is comprised of rocks and rain that fall at random speeds and a bucket that is movable by using the right and left arrow keys. When the user catches the rain drops the score increases and when the user catches a rock the game is over. The game over state allows the user to return to the home page, try again, and view their score. The home page at the start of my game is where the user can access the game and the instructions. Specifically, the game panel deals with the user interface as well as all of the different object interactions. The object manager deals with the objects appearing on the screen. Finally, each object has its own class that creates the object and its speed.

4. Technical Implementation

One aspect of my game that I am particularly proud of is the part that deals with the rain and rocks moving at random speeds. This part of my game makes my game more interesting and challenging to play. After I added the rocks and rain to my game by using array lists, I went into each specific rain and rock class and changed the speed to a random number. This number could be anywhere from 1 to 10 so the objects would not move too fast or slow. After that, I only made the rocks and rain travel in the y axis so they would just move up or down, not right or left.

5. Most Challenging Part

The most challenging part of this project was getting the bucket to not move off of the screen. I was stuck on this problem for a few classes and I asked my teacher Matt for help but, I also tried to fix some of the problems myself. All of the problems that this section of my code had were very minor. For example, I didn't put the bracket in the right location and my numbers were off in my if statements.

6. Future Fixes and Updates

In the future, I would find clearer photos to use as my background and I would have different levels and obstacles that the user would have to go through.
